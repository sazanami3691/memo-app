# Codex作業ルール

このリポジトリでCodexが作業する際は、以下のルールに従う。

## 作業ブランチ

- このリポジトリでCodexが使用するブランチは `codex-work` とする。
- 作業開始前に現在のブランチを確認する。
- `codex-work` 以外の場合は、ファイルを変更せずユーザーへ報告する。
- `main` には直接コミット・プッシュしない。

## 作業前の確認

- 作業開始前に `git status` を確認する。
- ユーザーが作成した未コミットの変更を削除・上書き・破棄しない。
- 関係のないファイルを変更しない。

## 実装と検証

- 依頼された範囲だけを変更する。
- 利用可能なテストや構文チェックがあれば、変更後に実行する。
- テストに失敗した場合は自動プッシュせず、原因をユーザーへ報告する。
- 秘密情報、`.env`、ログ、キャッシュ、一時ファイル、ユーザーが作成したメモデータをコミットしない。

## 自動コミットとプッシュ

- 依頼された変更が完了し、確認やテストに問題がなければ、`git status` と `git diff` で変更内容を確認する。
- 今回の作業で変更したファイルだけを個別に `git add` する。
- 関係のない既存変更はコミットに含めない。
- 内容が分かる簡潔なコミットメッセージを付けてコミットする。
- コミット後、`origin` の `codex-work` ブランチへ自動的にプッシュする。
- 変更がない場合は、空のコミットやプッシュを行わない。

## 禁止事項

- `main` への直接プッシュ
- force push
- ユーザーの許可がないブランチの削除
- ユーザーの許可がないマージやPull Request作成
- 競合の強制的な解決
- `git reset --hard` などによる変更の破棄

## 問題発生時

- Git競合、認証エラー、通信エラー、テスト失敗が発生した場合は、無理に処理を続けずユーザーへ報告する。
- 作業完了時に、変更したファイル、実行した確認、コミットID、プッシュ先ブランチを報告する。
